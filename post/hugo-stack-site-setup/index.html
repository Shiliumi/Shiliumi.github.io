Hugo + Stack 博客搭建记录：从项目初始化到本地预览

这篇文章记录本站基础搭建阶段做过的主要事情。它不是一份完全通用的教程，而是一次实际工程过程的复盘：目标是把 Stack 主题从默认示例站点改造成一个可以继续维护的个人博客。

选择 Hugo Extended 和 Stack

本站使用 Hugo Extended 作为静态站点生成器，主题选择 Stack。Hugo 的优势是构建速度快、部署结果简单，最终产物就是静态文件；Stack 的优势是文章列表、侧边栏、归档、搜索、分类和标签这些博客核心功能已经比较完整。

本地开发时主要依赖 Hugo server：

hugo server -D --renderToMemory --bind 127.0.0.1 --port 1313

-D 用于显示草稿文章，--renderToMemory 让预览过程尽量不污染输出目录。正式构建时再使用 Hugo 的构建命令生成静态页面。

基础配置

基础配置主要放在 hugo.toml 中。本站首先完成了几项全局设置：

  • 设置站点标题为“小翟的个人博客”
  • 设置默认语言为中文
  • 启用 enableGitInfo
  • 配置 frontmatter.lastmod
  • 设置 Stack 主题
  • 设置浏览器标签页图标

其中更新时间配置比较关键：

enableGitInfo = true

[frontmatter]
lastmod = [':git', ':fileModTime', ':default']

这样文章更新时间会优先读取 Git 提交时间。如果没有 Git 信息，再回退到本地文件修改时间，最后才使用默认时间。这个配置后续配合 GitHub Actions 部署，可以让线上页面也显示更接近真实变更历史的更新时间。

中文导航

主题默认菜单更偏示例站点，所以本站将左侧导航统一成中英双语形式：

  • 主页 | Home
  • 分类&标签 | Cats&Tags
  • 归档 | Archives
  • 搜索 | Search
  • 相关 | Related
  • 技术链接 | Links

这样做有两个好处：中文用户能直接理解每一栏的作用，同时保留英文名称，方便和主题原本的功能概念对应。

头像和个人信息

Stack 的左侧栏会读取 params.sidebar 中的头像和简介配置。本站将头像放入 Hugo 的 assets 管线中，并配置为：

[params.sidebar]
avatar = 'img/avatar.avif'
subtitle = '欢迎来到我的网站，此博客由【Hugo + Stack】进行搭建，主要分享个人生活和学习笔记'

使用 assets 而不是随手放在任意目录，是为了让 Hugo 能稳定找到资源，并在构建时生成正确路径。

分类、标签和内容组织

文章放在 content/post 下，每篇文章通过 front matter 设置分类和标签。例如本站后续会将 Hugo 搭建记录统一归到 Hugo 分类下，并使用 博客搭建、Stack、静态站点 等标签做更细的索引。

分类页已经改成“上方分类、下方标签”的结构。这样内容多起来以后，分类用于表达主题方向，标签用于表达更细的技术点。

本地预览和检查

每次改动后，最基本的检查是先跑一次构建：

hugo --minify

如果模板、资源路径或 front matter 写错，构建阶段通常会先暴露问题。构建通过后，再打开本地服务检查首页、文章页、分类页、归档页和搜索页是否都能正常工作。

小结

基础搭建阶段的重点不是一次性加很多功能，而是先把博客从主题演示站点变成自己的站点。标题、头像、导航、分类、联系方式、更新时间和本地构建流程都稳定以后，再继续叠加播放器、PJAX、动态背景、Live2D 等个性化功能会更稳。
