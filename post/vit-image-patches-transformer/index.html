Vision Transformer：把图像切成词以后发生了什么

重读 ViT 论文，理解图像 patch、class token、位置编码和大规模预训练对视觉 Transformer 的意义。

Vision Transformer 这篇论文的核心动作非常大胆：把图像切成固定大小 patch，把每个 patch 当成一个 token，然后几乎原封不动地使用 Transformer encoder 做图像分类。它看上去简单，甚至有点“偷懒”，但真正的结论并不简单：当预训练数据足够大时，Transformer 可以在视觉任务上摆脱 CNN 的强归纳偏置。

图：原论文 Figure 1，展示 ViT 如何把图像 patch 投影为 token 序列，并送入 Transformer encoder。这里直接从论文 PDF 裁取。

为什么这件事当时不算显然

在 ViT 之前，CNN 几乎是视觉主干的默认选择。卷积有强归纳偏置：局部连接、权值共享、平移等变性。这些偏置非常适合图像，也让模型在中等规模数据上更容易泛化。Transformer 在 NLP 里很强，但文本 token 天然是一维序列；图像是二维网格，直接套 Transformer 会不会太浪费？会不会因为缺少卷积偏置而学不动？

ViT 的答案有条件：如果只在 ImageNet-1k 这样的中等数据上从头训练，ViT 不一定比同规模 CNN 好；但如果在 JFT-300M 这类超大数据上预训练，再迁移到下游视觉任务，ViT 可以达到甚至超过强 CNN。也就是说，论文不是简单宣称“Transformer 比 CNN 强”，而是在说：数据规模足够大时，弱归纳偏置不再一定是劣势。

这个结论很重要。CNN 把很多视觉先验写死在结构里，样本效率高；ViT 让模型从数据里学更多结构，早期更吃数据，但扩展性强。

Figure 1：把图像变成序列

原论文 Figure 1 画得很清楚。输入图像被切成固定大小 patch，比如 16x16。每个 patch 展平后通过线性层投影成 D 维 embedding。然后在序列前加一个可学习的 class token，再加位置编码，送入标准 Transformer encoder。最后取 class token 的输出做分类。

这里最关键的不是 Transformer encoder 本身，而是 patch embedding。它相当于把 H x W x C 的图像变成长度为 N = HW / P^2 的 token 序列。patch size P 越小，序列越长，空间细节越多，但注意力计算更贵；P 越大，序列更短，计算省，但细粒度信息损失更多。ViT-B/16 里的 /16 指的就是 patch 大小。

class token 则借鉴了 BERT 的 [CLS]。它不对应某个图像块，而是在多层 self-attention 中聚合全局信息，最后用于分类。位置编码用来告诉模型 patch 的空间位置。没有位置编码，patch 序列就会丢掉二维布局。

ViT 到底少了什么视觉先验

ViT 和 CNN 最大的差异，是它没有把局部性和权值共享写得那么强。CNN 的同一个卷积核会在整张图上滑动，天然偏向局部纹理；ViT 的每个 patch token 经过线性投影后，第一层 self-attention 就可以看到所有 patch。它更自由，但也更需要数据告诉它哪些局部结构可靠。

这解释了论文里的一个重要现象：在较小数据集上，ViT 表现不如 ResNet 等 CNN；在大规模预训练后，ViT 的表现明显提升。归纳偏置和数据规模之间有交换关系。强偏置帮助小数据泛化，弱偏置给大数据扩展留下空间。

论文还比较了 hybrid 模型，也就是先用 CNN 提取特征，再把特征图切成 token 输入 Transformer。这个方向说明作者并不是把 CNN 一脚踢开，而是在测试不同程度的视觉先验。后来很多模型也沿着混合路线走过一段，比如 ConvStem、Swin Transformer 的局部窗口注意力等。

计算代价在哪里

自注意力的复杂度随 token 数平方增长。对图像来说，如果 patch 太小，token 数会迅速变多。224x224 图像用 16x16 patch 得到 196 个 patch，加 class token 是 197，计算还可控；如果改成 8x8 patch，token 数会变成 784，注意力成本大幅增加。

所以 patch size 是 ViT 里非常关键的旋钮。它不只是分辨率选择，而是在精度和计算之间做交易。论文中的 ViT-B/16、ViT-B/32、ViT-L/16 等命名，都反映了模型规模和 patch 大小的组合。更小 patch 通常更强，但也更贵。

这也解释了后续视觉 Transformer 的许多改进：Swin 用窗口注意力降低复杂度，PVT 做金字塔结构，DeiT 研究数据高效训练，MAE 用自监督重建提升预训练效率。ViT 打开了门，但第一版并不解决所有视觉计算问题。

实验结论别读反

ViT 论文最容易被读反的一点，是把它当成“CNN 已经没用了”。原文并没有这么简单。作者明确展示，在没有大规模预训练时，ViT 的表现并不一定好；而当预训练数据增大，ViT 的扩展性开始显现。它挑战的是“视觉模型必须依赖卷积主干”这个默认假设，而不是否认卷积在小数据和高效场景里的价值。

论文在 ImageNet、CIFAR、VTAB 等任务上报告迁移结果，并比较不同模型大小。真正值得看的趋势是：模型越大、预训练数据越大，ViT 越能发挥。这个趋势后来和大模型时代的经验合流：更通用、更少硬编码偏置的架构，往往在海量数据和算力下更有上限。

但这也意味着成本。ViT 的成功条件更依赖预训练资源。对于普通项目，直接从头训练一个 ViT 未必划算，使用预训练权重、蒸馏或更数据高效的变体通常更现实。

今天复现时最该动的旋钮

如果自己实现 ViT，最值得观察的是 patch size、embedding dim、depth、heads 和预训练数据。写代码并不难：切 patch、线性投影、加 class token 和位置编码、送入 Transformer encoder。难的是让模型在有限数据上泛化。

位置编码在微调高分辨率图像时也要处理。因为 patch 数变了，位置编码长度也变了，通常需要插值。这是很多复现细节里容易漏掉的一步。另一个细节是分类头：预训练时可能用 MLP head，微调时常换成线性头，论文里也提到这种设置差异。

ViT 最有启发的地方，是它把图像建模重新表述成序列建模。它没有发明复杂视觉模块，而是问了一个更根本的问题：如果把图像切成 token，标准 Transformer 能走多远？答案是，在足够数据面前，能走得很远。
